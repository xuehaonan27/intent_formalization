PARAMS: addr: *mut u8, len: usize, block_size: usize, result: Result<Slab, Error>
BODY:
```rust
// --- Concrete input ---
assume(addr as usize == 0x1000);
assume(block_size == 1024);
assume(len == 3072);  // 3072 / 1024 = 3 blocks exactly
// Ideal: end_addr = 0x1000 + 3*1024 = 0x1000 + 3072 = 0x1C00

// --- Concrete output (with bad property) ---
assume(result is Ok);
let slab = result.unwrap();
assume(slab@.block_size == 1024);
assume(slab@.start_addr == 0x1400);  // After 1 index block (1024 bytes)
// Bad property: end_addr covers only 1 data block instead of 2
// Ideal: end_addr = 0x1C00, actual usable = [0x1400, 0x1C00) = 2 data blocks
// We use end_addr = 0x1800, losing 1 entire 1024-byte block
assume(slab@.end_addr == 0x1800);
assume(slab@.allocated_addrs == Set::<usize>::empty());
assume(slab@.free_addrs == Set::<usize>::empty().insert(0x1400usize));
```
REASON: With large block_size, each wasted block is substantial; spec allows losing 1024+ bytes that the body implementation would correctly utilize
