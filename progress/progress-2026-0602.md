### 2.2 Other spec obligations (per [`spec-accountability/spec-obligations-overview-1.0.md`](../spec-accountability/spec-obligations-overview-1.0.md))

Determinism (obligation **1.1**) is the line this repository has
operationalised end-to-end. The accountability overview defines five
further obligations under the same umbrella — a spec that the
verifier accepts is not necessarily a spec that *should* have been
written.

Notation used in the table:

* `S` — a formal specification (the conjunction of its clauses).
* `C` — a single clause of `S`.
* `A(S)` — the *admissible-implementation set*: every implementation
  that the verifier would accept against `S`.
* `eq_f` — the equivalence relation on observable outcomes that
  defines "two impls behave the same".
* `R = {r_1, …, r_m}` — the set of atomic, non-overlapping
  requirements decomposed from the user's natural-language intent.
* `M ∈ {0,1}^{|R|×|S|}` — the binary coverage matrix with
  `M[i, j] = 1` iff clause `c_j` alone entails requirement `r_i`.
* `M_n` — the per-node analogue of `M` at a non-leaf call-graph
  node `n`: rows are atoms of `n`'s spec, columns are atoms of
  `n`'s direct callees' specs.

| # | Obligation | Group | Definition |
|---|---|---|---|
| 1.1 | Determinism | intent-free | **Done** — `S` shrinks `A(S)` to a single point modulo `eq_f`: every pair of admissible impls produces `eq_f`-equivalent observations on every input. |
| 1.2 | Attributive Non-Vacuity | intent-free | No clause of `S` is semantically redundant: removing any clause `C` strictly enlarges the admissible-impl set, so every clause carries weight in defining `A(S)`. |
| 2.1 | Intent Traceability & Drift | intent-related | Per method, two checks on the binary coverage matrix `M ∈ {0,1}^{\|R\|×\|S\|}` between atomic intent requirements `R` and spec clauses `S`: **(static)** `M` has no row-zero (every intent atom is covered by some clause) and no col-zero (every clause services some intent atom); **(drift)** across consecutive snapshots labelled by edit channel (`intent-led` / `spec-led` / `joint`), the diff of `M` matches the user-declared blast radius — no unannounced spillover and no stale clauses left behind by an intent edit. |
| 2.2 | Compositional Sufficiency | intent-related | At every non-leaf call-graph node `n`, the matrix `M_n` mapping parent spec atoms to callee spec atoms has no row-zero (parent clause unsupported by any callee) and no col-zero (callee guarantee never consumed). |
| 2.3 | Correctness | intent-related | `A(S)` contains only implementations the user actually intended — no "correct under the spec but wrong from the user's view" impls slip through. (deferred — structurally hardest) |
| 3.1 | Assumption Accountability | global | Every claim made by 1.1 / 1.2 / 2.1 / 2.2 traces in the repository-wide Trust Provenance Graph to declared source nodes with `{rationale, owner, justification_ref, last_reviewed}`; no hidden, orphan, or unjustified sources. |
