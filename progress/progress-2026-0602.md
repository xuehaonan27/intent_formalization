### 2.2 Other spec obligations (per [`spec-accountability/spec-obligations-overview-1.0.md`](../spec-accountability/spec-obligations-overview-1.0.md))

Determinism (obligation **1.1**) is the line this repository has
operationalised end-to-end. The accountability overview defines five
further obligations under the same umbrella — a spec that the
verifier accepts is not necessarily a spec that *should* have been
written:

| # | Obligation | Group | Definition |
|---|---|---|---|
| 1.1 | Determinism | Spec↔Impl | **Done** — `S` shrinks `A(S)` to a single point modulo `eq_f`: every pair of admissible impls produces `eq_f`-equivalent observations on every input. |
| 1.2 | Attributive Non-Vacuity | Spec↔Impl | Every clause `C` of `S` is load-bearing in defining `A(S)`: deleting `C` strictly enlarges `A(S − C) ⊋ A(S)`. |
| 2.1 | Intent Traceability & Drift | Spec↔Intent | Per method, the binary coverage matrix `M ∈ {0,1}^{|R|×|S|}` between atomic intent requirements `R` and spec clauses `S` has no row-zero (uncovered intent) and no col-zero (clause without intent), and stays consistent across intent/spec edits. |
| 2.2 | Compositional Sufficiency | Spec↔Intent | At every non-leaf call-graph node `n`, the matrix `M_n` mapping parent spec atoms to callee spec atoms has no row-zero (parent clause unsupported by any callee) and no col-zero (callee guarantee never consumed). |
| 2.3 | Correctness | Spec↔Intent | `A(S)` contains only implementations the user actually intended — no "correct under the spec but wrong from the user's view" impls slip through. (deferred — structurally hardest) |
| 3.1 | Assumption Accountability | Meta | Every claim made by 1.1 / 1.2 / 2.1 / 2.2 traces in the repository-wide Trust Provenance Graph to declared source nodes with `{rationale, owner, justification_ref, last_reviewed}`; no hidden, orphan, or unjustified sources. |

2.2 and 3.1 stay as drafted plans, activating once 1.2 and 2.1
produce concrete artifacts to feed them. 2.3 is acknowledged and
held.
