# P5 完成 review：结构化审计标签 + `returns` 提取（2026-07-21）

## P5 — 结构化审计标注（HANDOFF §13 P5）

A/B/C 审计标签从 Markdown 落入机器可读元数据。`result.json` 新增字段：

- `audit_label`：六个桶之一——`complete` / `complete_tool_gap` /
  `incomplete_permitted` / `incomplete` / `unsupported*` / `unknown`
  （`*` 含 `unsupported_mut_ref_return`、`verus_error` 等 status 原值）；
- `audit_note`：该目标的审计说明（C 类理由、机制说明、杂项备注）；
- 前序 P3 字段（`permitted` / `permitted_reason` / `quotient`）保留。

关键设计：

- **标注在每一个出口路径生效**（`_finalize` 统一处理，包括
  unsupported / verus_error / no_ensures 的早退路径）；
- **C 类（7 个）的 `incomplete` 标签是人工审计确立的**——没有机器 sat
  witness；`audit_note` 明确写出这一点，不会与求解器判定混淆；
- A 类（8 个 P2 目标）验证通过后标记为 `complete_tool_gap`，`AUDIT_TOOL_GAP`
  记录了各自的修复机制；商等号构造器标记为 `complete` 并带 `quotient` 字段，
  与普通 complete 可区分；
- `SUMMARY.md` 新增 Audit 列与 audit-label 计数行，聚合器可直接出六个桶的报表。

## 零散项 1 — `returns` 子句提取

extractor 此前只取 `ensures`，两个 `read` 目标（`pcell::read@224`、
`pcell_maybe_uninit::read@234`）因使用 `returns` 子句被判 `no_ensures`。
现在 `_extract_clauses` 同时收集 `returns_clause`，并在 `extract_spec` 中
按 Verus 语义下降为 `ensures <返回绑定名> == <expr>`（绑定名来自
`named_return_type`，管线本已捕获 `result_binding`）。

- 两目标在两个快照上均翻 `complete`（postcondition 恰好钉死返回值）；
- vstd 共 49 个 `returns` 子句点，今后全部进入确定性检查覆盖；
- extractor 自测全过，无回归。
