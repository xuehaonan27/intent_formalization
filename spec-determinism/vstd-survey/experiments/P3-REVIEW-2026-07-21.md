# P3 完成 review：permitted nondeterminism 显式编码（2026-07-21）

## 设计原则（HANDOFF §13 P3）

> 不允许用全局 `equal == true` 吞掉 B 类；每个目标必须有显式记录的商等号
> 或 permitted 原因。

实现（`run_determinism.py`，复用 P2 的覆写机制）：

- **商等号构造器（8 个）**：`EQUAL_FN_OVERRIDES` 增加条目，只去掉身份合取项
  （CellId / 分配地址 / 锁实例），保留内容/谓词比较。判定转为 `complete`，
  `result.json` 记录 `quotient` 名称与 `permitted_reason`。
- **关系型非确定（4 个）**：无任何商等号能使结果唯一。分类由
  `ok_inconclusive` 改为 **`incomplete_permitted`**，`result.json` 记录
  `permitted: true` 与原因。

## 语义依据

商等号目标之所以能验证，是因为 ensures 恰好钉住商内的全部内容：
`pc_mu::{empty,new}` 钉 `mem_contents() == Uninit/Init(v)`；`pcell::new` 钉
`value() == v`；`simple_pptr::empty` 钉 `is_uninit()`；`rwlock::new` 钉
`pred() == pred`。被忽略的恰好只是有意自由的 identity。
