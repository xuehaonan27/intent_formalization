## 背景

HANDOFF §13 P2 要求把 `UNKNOWN-AUDIT-2026-07-15.md` 人工验证过的 A 类修复
自动化。实现方式是 runner 级的数据驱动机制（`run_determinism.py`），不改动
pipeline 内核：

- `EQUAL_FN_OVERRIDES`：按 `(module, function, line)` 键（支持 `"*"` 通配行号
  以跨快照复用）替换生成的 equal-fn；
- `PROOF_HINTS`：按同样键在 det proof fn 末尾注入证明语句，可选把结果参数
  改写为 `tracked`（proof fn 参数默认 spec mode，无法调用 tracked 引理）。

## 实现过程中的三个 Verus 语义细节（对后续写证明提示很重要）

1. det fn 已是 proof fn，体内不能再写 `proof { }` 块（报 "proof blocks
   inside spec code ... only for spec functions with decreases"）；
2. proof fn 参数默认 **spec mode**——调用 tracked 引理/`Tracked::borrow`
   需要把结果参数显式声明为 `tracked`（spec fn 不能接受 tracked 参数，
   所以改写只作用于 proof fn 签名）；
3. 引理的 precondition 若只在 det fn 的 ensures 前件里，则证明体中不可见
   （前件不是上下文假设）；需要显式 `assume(前件事实)` 再调引理——
   det fn 的对外契约不变，语义保持健全。
